# 文章脉络建议

## 1. Introduction

这一节要先把问题打出来：

现有大模型在代码生成、视觉理解、3D reasoning 上进步很快，但 CAD 这个方向的 benchmark 仍然偏简单。很多数据集主要包含 box、cylinder、simple extrusion、simple boolean 这类几何体，不能代表真实工业 CAD。

然后引出你们的核心观察：

> 工业 CAD 的难点不只是生成一个形状，而是生成 **可执行、可编辑、参数化、符合工程约束、包含复杂 operations 的 CAD program**。

可以强调三个 gap：

1. **Industrial relevance gap**
   现有 benchmark 缺少符合工业标准或工程语义明确的零件，比如 twist drill、handwheel、involute gear。

2. **Operation complexity gap**
   现有 CAD 数据过度依赖简单 extrude/cut，缺少 revolve、sweep、helix、fillet、chamfer、array、gear tooth construction 等复杂操作。

3. **Evaluation gap**
   只用 IoU/CD 不够，因为 CAD code 是否可执行、是否用了正确 feature、是否满足标准参数关系同样重要。

最后列贡献：

```text
1. We introduce an industrial CAD benchmark built from standard-inspired
   parametric part templates.

2. We design multiple evaluation tasks covering CAD generation, CAD understanding,
   and CAD editing.

3. We benchmark open and proprietary MLLMs and show that current models still
   struggle with engineering-grade CAD reasoning.

4. We introduce operation-level metrics, including rare operation recall, to
   measure whether models learn advanced CAD construction patterns.

5. We provide SFT/RL training baselines showing that CAD-specific training improves
   operation coverage, but does not fully solve compositional generalization.
```

## 2. Related Work

这里不要写太散，分三块。

### 2.1 CAD and 3D Generation Benchmarks

讲 DeepCAD、Fusion360、Text2CAD、CAD-MLLM 类工作。重点不是贬低别人，而是说它们分别解决了不同问题，但缺少你们这个维度：

| 以前工作                 | 不足                                                                         |
| -------------------- | -------------------------------------------------------------------------- |
| DeepCAD / Fusion360  | 更偏 CAD sequence 或 CAD history，不一定是大模型 multimodal code generation benchmark |
| Text2CAD 类           | text-to-CAD，但工业标准和复杂 ops 覆盖不足                                              |
| image/3D-to-CAD 类    | 多数更关注几何重建，少关注 executable parametric CAD code                               |
| general 3D benchmark | 不评估 CAD operations 和工程约束                                                   |

这里对应你的 **Table 1：各个数据集对比**。

### 2.2 MLLM Evaluation for Code / 3D / Engineering Reasoning

讲大模型会写代码，但 CAD code 不只是普通 Python code。它要有空间推理、几何操作、工程参数约束。

### 2.3 Training MLLMs for Domain-Specific CAD

讲 SFT/RL 作为 domain adaptation，不是你们的主算法贡献。提前埋伏笔：

> 我们训练 reference models，是为了研究工业 CAD 数据能教会模型什么，而不是提出一个通用 SOTA 模型。

## 5. Tasks and Evaluation Protocol

这一节很重要，因为你们不是只有一个 generation 任务。

你可以把 benchmark 分成 3 个 task groups。

---

### 5.1 CAD Generation

输入：image / point cloud / image+point cloud
输出：executable CadQuery code

指标：

```text
Execution Pass
IoU
Chamfer Distance
Feature F1
Operation Recall
Rare Operation Recall
Standard Compliance
```

---

### 5.2 CAD Understanding

输入：render / point cloud / code / metadata context
输出：part type、参数、功能、feature 问答

例子：

```text
What is the likely part family?
How many teeth does the gear have?
Which operation creates the central bore?
Does the drill contain helical flutes?
```

指标：

```text
accuracy
exact match for parameters
semantic correctness
```

---

### 5.3 CAD Editing

输入：原始 CAD code 或 shape + edit instruction
输出：edited executable code

例子：

```text
Increase the bore diameter while preserving the tooth count.
Change the handwheel from 3 spokes to 5 spokes.
Add a keyway to the shaft.
```

指标：

```text
edit success
non-target preservation
execution pass
geometry consistency
feature correctness
```

---

这一节最后强调：

> We evaluate not only whether the generated geometry looks similar, but also whether the program uses the intended CAD construction operations and preserves engineering constraints.

## 12. Limitations

这一节主动讲：

1. 数据是参数化生成的，不完全等于真实工业 CAD 公司数据；
2. ISO 标准可能是 standard-inspired，不一定覆盖所有真实制造约束；
3. SFT/RL reference model 不是通用 SOTA；

# 你这篇最强的 narrative

最终这篇文章要让 reviewer 记住一句话：

> **Existing CAD benchmarks ask whether models can make simple shapes. Our benchmark asks whether models can generate executable, editable, operation-rich CAD programs for standard-compliant industrial parts.**

SFT/RL 的结论是：

> **CAD-specific training teaches models more rare operations, but current MLLMs still fail at compositional industrial CAD reasoning.**

这个 story 比“我们在自己 benchmark 上 SOTA”稳很多。
