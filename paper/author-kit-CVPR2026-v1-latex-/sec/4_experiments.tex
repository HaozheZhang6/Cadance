\section{Experiments}
\label{sec:exp}

\subsection{Evaluation Setup}

We evaluate the pipeline as a data engine. The primary question is: how many high-precision program--geometry pairs does the system produce, at what verification rate, and what is the contribution of each component (auto pipeline, cascade, repair, manual checking)?

\noindent\textbf{Dataset:} Ground-truth STEP files from Fusion360 Gallery~\cite{fusion360gallery} (approximately 13k parts available; we ran batches over approximately 6k). Synthetic parts generated by our primitive generators. Total 26 batch runs across all providers.

\noindent\textbf{Metric:} 3D IoU via OCCT (threshold $\tau = 0.99$). All 1,646 accepted pairs satisfy this threshold by construction.

\noindent\textbf{Providers:} OpenAI \texttt{gpt-5.2} (primary), Codex CLI \texttt{gpt-5.3-codex} (secondary), GLM \texttt{glm-4.6v} (tertiary), Claude Code manual (last resort).

\subsection{Qualitative Case Studies}

Figure~\ref{fig:autopasses} shows representative parts accepted on the first auto attempt, spanning multiple complexity classes: a complex arc-profile Fusion360 part, a radial 8-fin assembly, a 6-wedge ring segment, and a U-bracket with large-radius arcs. All achieve IoU~$= 1.000$. These cases demonstrate that the auto pipeline handles genuine geometric complexity without repair.

Figure~\ref{fig:repairs} shows cases where auto-generation failed and the manual check skill recovered the correct program. Each row shows the ground-truth ISO view, the initial generated geometry (IoU $< 0.90$), and the fixed geometry (IoU $\geq 0.99$). Across cases, the dominant failure modes are XZ-plane sign errors, YZ-plane axis swaps, missing inner loops, and \texttt{radiusArc} direction selection. After targeted repair, all cases reach IoU $\geq 0.99$.
