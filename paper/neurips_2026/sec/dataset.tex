\section{The BenchCAD Dataset}
\label{sec:dataset}

\paragraph{Design principles.} BenchCAD rests on four principles. \emph{(P1) Execution as ground truth}: every record's CadQuery source is run inside a sandboxed environment, the resulting STEP voxelised at $64^3$, and matched against a reference solid by a 6-orientation rotation-invariant 3D IoU $\geq 0.99$~\citep{rot_iou} --- ruling out the silent-failure mode of corpora that merely parse code without re-execution. \emph{(P2) Standard-table anchoring}: where a part has an industrial counterpart, parameters sample from real specification tables (ISO 22 V-belt cross-sections, DIN 338 twist-drill diameters, ISO 23509 bevel-gear pitch--module relations); 49\% of BenchCAD families (52/106) are standard-anchored, drawing from 47 unique ISO/DIN/EN/ASME/IEC codes (full code list in Appendix~\ref{app:standards}). \emph{(P3) Family-level taxonomy}: records are grouped into 106 named families (\texttt{coil\_spring}, \texttt{helical\_gear}, \texttt{twisted\_drill}, $\ldots$), each implemented by a small Python module exposing a typed parameter schema, sampler, validator, and deterministic builder --- making coverage measurable and per-family analysis trivial. \emph{(P4) Operation breadth}: the dataset exercises 45+ distinct CadQuery operations spanning sketch primitives, contour drawing, workplane manipulation, sketch arrays, 3D operations (extrude, twist-extrude, revolve, loft, sweep), Booleans, hole variants, and edge finishing (Appendix~\ref{app:ops}).

\paragraph{CadGen pipeline.} The data construction pipeline, \textbf{CadGen}, runs four stages per family $\times$ difficulty bucket: (1) deterministic parameter sampling from the JSON schema under standard-table constraints; (2) program emission, where the family's \texttt{make\_program} returns an op sequence consumed by a builder that produces both Python CadQuery source and a CadQuery \texttt{Workplane} object aligned by construction; (3) sandboxed execution and verification (re-execute code, export STEP, voxelise, compute rotation-invariant IoU against the builder's direct output, quarantine on IoU $<0.99$, execution failure, $>$30\,s timeout, or zero/inverted volume); and (4) rendering to four canonical views (front/right/top/iso) at $512^2$ with VTK. From a 17{,}968-sample April 2026 batch, 17{,}900 records (99.6\%) passed all checks.

\paragraph{Splits, taxonomy, hosting.} The release contains \emph{BenchCAD-Synth} (17{,}900 verified procedural parts, primary evaluation set), \emph{BenchCAD-Edit} (748 curated edit pairs across dimensional, additive, subtractive, and multi-step categories), \emph{BenchCAD-Lite} (530-sample stratified subset for fast leaderboards), \emph{BenchCAD-Fusion360} and \emph{BenchCAD-DeepCAD} (verified OOD subsets), and \emph{BenchCAD-GenCAD} (162{,}000 image$\to$code training pairs)~\citep{GenCAD2025}. Each family supports easy/medium/hard tiers defined by parameter complexity. Every release ships a Croissant 1.0 metadata file~\citep{Croissant2024} validated by the public checker, hosted on Hugging Face under \texttt{BenchCAD/cad\_bench} with verification-pipeline source under MIT and data under CC-BY-4.0. Full datasheet (Appendix~\ref{app:datasheet}) and per-family schemas accompany the release.
