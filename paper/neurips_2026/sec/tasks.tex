\section{Benchmark Tasks and Capability Decomposition}
\label{sec:tasks}

\subsection{The Image$\to$Code Capability Chain}
\label{sec:capability}

BenchCAD's four task categories correspond to stages of practising CAD-engineering work: an engineer reads an existing drawing or part (\texttt{img2cq} reconstruction), interrogates dimensions and operations on either the rendered geometry or the source program (\texttt{qa\_img}/\texttt{qa\_code}), and modifies an existing parametric program in response to a request (\texttt{edit\_img}/\texttt{edit\_code}). A single end-to-end metric on an image-to-code task reports \emph{whether} a model succeeded but not \emph{which capability} failed. We decompose the image$\to$code task into three causal links (Table~\ref{tab:capability}): \emph{$C_1$ Visual + 3D Reconstruction} ($f_{C_1}\!:\,\text{multi-view image} \to \text{3D shape}$); \emph{$C_2$ Geometric$\to$Parametric Abstraction} ($f_{C_2}\!:\,\text{3D shape} \to \{(p_i,v_i)\}$, identifying which dimensions are bores, which are wall thicknesses, which are ratios); and \emph{$C_3$ Parametric$\to$Code Synthesis} ($f_{C_3}\!:\,\{(p_i,v_i)\} \to \text{CadQuery code}$). BenchCAD's tasks are designed so each exercises a known subset of $\{C_1, C_2, C_3\}$ and so score \emph{differences} across paired tasks isolate the bottleneck capability. The key contrast --- the gap between \texttt{qa\_img} and \texttt{qa\_code} on matched prompts --- measures the cost of replacing direct code access with visual perception (i.e.\ $C_1$ alone) and is the central diagnostic finding (\textbf{Vision Bottleneck}, \S\ref{sec:exp_capability}). Three further interpretable contrasts apply: \texttt{qa\_code} (low absolute) $\Rightarrow$ $C_2$ bottleneck; \texttt{img2cq}$-$\texttt{edit\_code} large negative $\Rightarrow$ $C_3$ + full-stack composition; \texttt{edit\_img}$-$\texttt{edit\_code} large negative $\Rightarrow$ Vision Bottleneck preserved on edit.

\subsection{Task Definitions and Scoring}

\paragraph{Five tasks, four categories.} \texttt{img2cq} (image-to-code): four canonical orthographic views $\to$ CadQuery code; scored by $\mathrm{Pass}_{\mathrm{IoU}\geq 0.5}$ under the rotation-invariant cube symmetry group, with mean Chamfer distance, invalidity ratio, and Feature-F1 as auxiliary metrics. \texttt{qa\_img}/\texttt{qa\_code} (visual / code-conditioned numeric QA): a 2{,}400-question per-family bank asking only ratios and integer counts (never absolute mm) for scale invariance, scored by relative error $\leq 5\%$ for ratios and exact match for integers; the paired design (each question evaluated under both modalities) directly isolates $C_2$ from $C_1$. \texttt{edit\_img}/\texttt{edit\_code} (image-conditioned / text-only editing): a 748-pair instruction-guided edit set spanning dimensional/additive/subtractive/multi-step categories, scored as Pass@1 = $\mathbb{1}[\mathrm{IoU}(\text{edited},\text{target})\geq 0.99 \wedge \mathrm{Preserve}(\text{edited},\text{orig})\geq 0.99]$ requiring both target success and non-target preservation; we additionally report the disaggregated Edit Leakage rate (\S\ref{sec:exp_edit}). The headline aggregate is the macro-mean across the five tasks. Question bank construction, edit-pair protocol, rotation-invariant IoU, and a scale-invariance ablation appear in Appendices~\ref{app:qbank}, \ref{app:edit_protocol}, \ref{app:iou}, and \ref{app:ablation} respectively.
